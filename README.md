# ЛР3 — Генераторы псевдослучайных чисел

Лабораторная работа по дисциплине, посвящённой генерации и тестированию
псевдослучайных чисел (ПСЧ). Реализованы **три собственных генератора ПСЧ**,
проведена проверка выборок на равномерность (критерий χ²) и случайность
(пять тестов NIST), а также измерена скорость генерации в сравнении со
стандартным генератором языка `std::mt19937_64`.

## Реализованные методы

| Обозн. | Класс | Основа | Кратко |
|--------|-------|--------|--------|
| A | `ChronoSplit` | время + микшер splitmix64 | затравка из таймера в наносекундах, биты перемешиваются умножениями и сдвигами |
| B | `CustomLCG` | линейный конгруэнтный генератор | `state = A·state + C mod 2⁶⁴`, наружу отдаются перемешанные старшие половины двух шагов |
| C | `CustomXorshift` | xorshift128+ | состояние из двух 64-битных слов, период 2¹²⁸−1, собственные сдвиги a=23, b=17, c=26 |

Все три генератора реализованы самостоятельно на C++ без использования
стандартных библиотек ПСЧ (`std::mt19937_64` подключается только как эталон
для сравнения скорости).

## Что проверяется

1. **Статистика выборок** — для каждой выборки считаются среднее, СКО и
   коэффициент вариации; они сравниваются с теоретическими значениями для
   равномерного распределения на `[0, RANGE)`.
2. **Равномерность** — критерий χ² (диапазон делится на `BINS` интервалов,
   статистика сравнивается с критическим значением при df = BINS−1).
3. **Случайность** — пять тестов NIST SP 800-22: частотный (monobit), тест
   серий (runs), блочный частотный (block frequency), тест пар бит (serial),
   накопленные суммы (cumulative sums). p-value вычисляется через неполную
   гамма-функцию / `erfc`.
4. **Скорость** — замер времени генерации от 1 000 до 1 000 000 элементов
   тремя методами и стандартным `std::mt19937_64`, построение графиков.

## Параметры эксперимента

Задаются константами в начале `main.cpp`:

| Параметр | Значение | Смысл |
|----------|----------|-------|
| `RANGE` | 10000 | диапазон значений `[0, RANGE)` (≥ 5000 по ТЗ) |
| `N` | 1000 | элементов в одной выборке (≥ 1000 по ТЗ) |
| `SAMPLES` | 20 | число выборок на метод (≥ 20 по ТЗ) |
| `BINS` | 100 | число интервалов для χ² |

## Сборка и запуск

Требуется компилятор с поддержкой C++17.

```bash
g++ -O2 -std=c++17 main.cpp -o prng
./prng
```

При запуске программа:
- печатает статистику и значение χ² по каждой из 20 выборок для всех трёх методов;
- печатает сводную таблицу по пунктам 3–4 и теоретические ориентиры;
- прогоняет пять тестов NIST для каждого метода (вердикт PASS / FAIL по порогу α = 0.01);
- замеряет скорость генерации и сохраняет результаты в `data/timing.csv`.

> Строка `system("chcp 65001 > nul")` нужна только для корректного вывода
> кириллицы в консоль Windows и на Linux/macOS безвредна.

## Краткие результаты

- Все три генератора дают среднее ≈ 5000, СКО ≈ 2887 и коэффициент вариации
  ≈ 57.7 %, что совпадает с теорией для равномерного распределения на `[0, 10000)`.
- По критерию χ² подавляющее большинство выборок признаны равномерными
  (ChronoSplit — 20/20, CustomLCG и CustomXorshift — по 19/20; единичные
  превышения порога ожидаемы при уровне значимости 0.05).
- Все три метода прошли все пять тестов NIST (все p-value ≥ 0.01).
- Собственные методы по скорости сопоставимы со стандартным `mt19937_64`
  и не уступают ему; самыми быстрыми оказались ChronoSplit и CustomXorshift
  (см. `timing_log.png`).
