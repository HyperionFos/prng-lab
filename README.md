# Лабораторная работа: Алгоритмы сортировок

**Вариант 21** — массив авиарейсов, прибывающих в аэропорт.

## Постановка задачи

Реализовать на C++ три алгоритма сортировки массива объектов с
многополевым сравнением, считать данные из CSV, замерить время работы на
наборах разной размерности, сравнить с `std::sort` и построить графики.

## Структура данных

Каждый рейс (`struct Flight`) описывается полями:
- номер рейса (`flightNumber`),
- авиакомпания (`airline`),
- дата прилёта (`arrivalDate`, формат `YYYY-MM-DD`),
- время прилёта по расписанию (`arrivalTime`, формат `HH:MM`),
- число пассажиров на борту (`passengers`).

## Правила сравнения

Операторы `<`, `>`, `<=`, `>=` перегружены и реализованы через единый
метод `compare`, который сравнивает поля в следующем приоритете:

1. дата прилёта — по возрастанию,
2. время прилёта — по возрастанию,
3. название авиакомпании — по возрастанию (лексикографически),
4. число пассажиров — **по убыванию**.

## Реализованные сортировки

- **Сортировка пузырьком** (Bubble sort) с флагом ранней остановки.
- **Шейкер** (Cocktail sort) — двунаправленный пузырёк.
- **Быстрая сортировка** (Quick sort) — Hoare partition + median-of-three pivot.
- **`std::sort`** из стандартной библиотеки (introsort) — для сравнения.

## Сборка и запуск

Требуется g++ с поддержкой C++20 (тестировалось на MinGW-w64 / UCRT64 16.1.0).

```bash
.\build.bat
.\sort_lab.exe
```

Программа генерирует датасеты размерности 100, 200, 500, 1000, 2000, 5000,
10000, 20000, 50000, 100000, сохраняет их в `data/dataset_<n>.csv`,
читает обратно и замеряет время каждой сортировки усреднением по 3
прогонам. Результаты пишутся в `data/timings.csv`.

Для построения графиков:
```bash
pip install matplotlib pandas
python scripts/plot_timings.py
```

## Результаты

### Линейная шкала

![Linear](data/plot_linear.png)

На линейном масштабе виден квадратичный взрыв пузырька и шейкера: на
50000 элементов они занимают около 75 секунд, тогда как quick и std
лежат фактически на оси X. При маленьких объемах данных разница между сортировками минимальна

### Логарифмическая шкала

![Log-Log](data/plot_log.png)

В log-log координатах степенные зависимости превращаются в прямые с
наклоном, равным показателю степени. Видно две пары почти параллельных
прямых: bubble/shaker с наклоном примерно 2 (O(n²)) и quick/std с наклоном примерно 1.1
(O(n log n)). std::sort стабильно быстрее учебного quick на 20–40% —
за счёт переключения на insertion sort на малых подмассивах и
страховки от плохого pivot в виде heapsort (introsort).

## Выводы

- Пузырёк и шейкер ведут себя практически одинаково на случайных данных. Преимущество шейкера проявляется на
  «почти отсортированных» массивах, чего в нашем эксперименте не было.
- На размерах до ~500 разница между алгоритмами незначительна — на
  малых данных можно использовать любой.
- Начиная с ~5000 элементов пузырёк становится практически непригоден:
  на 50000 уже ~75 секунд против ~40 миллисекунд у `std::sort` —
  разница в ~2000 раз.
- Эмпирические измерения подтверждают теоретические оценки сложности:
  при росте размера в 10 раз пузырёк замедляется примерно в 100 раз
  (n²), а std::sort — примерно в 12 раз (n log n).
- В реальном продакшене стоит использовать `std::sort` — он
  гарантированно O(n log n) в худшем случае (учебный quick на
  специально подобранных входах вырождается в O(n²)) и стабильно
  быстрее на практике.
